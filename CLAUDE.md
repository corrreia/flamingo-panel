Use Bun instead of Node.js (`bun install`, `bun run <script>`, `bunx <package>`).

Do NOT edit `.sql` files, generate migrations, or run migrations — database schema changes are handled manually.

Use Biome for linting/formatting (not ESLint/Prettier). Run `bun run check` before committing.

See [README.md](README.md) for project overview, setup, and scripts. See [ARCHITECTURE.md](ARCHITECTURE.md) for codebase structure and conventions.
